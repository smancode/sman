# SMAN 产品灵魂

## 核心价值

**可控可编辑**

大厂 AI 是黑盒 —— 你不知道它怎么想的，你不能改它的行为。

SMAN 不同：
- 所有 Skill 都是代码 —— 看得懂、改得了
- 所有配置都是文件 —— 版本控制、团队共享
- 所有数据都在本地 —— 你完全拥有
- 所有行为可定制 —— 适应你的工作流

> **一句话：这是你的 AI，不是租来的。**

---

## 使命

**让智能主动赋能用户**

## 痛点洞察

当前使用 AI 的最大痛点：**门槛太高**

- 哪怕很多程序员都用不起来
- 需要理解 prompt engineering
- 需要配置各种环境和工具
- 需要学习如何与 AI 协作
- 每次都要从零开始描述上下文

## 产品哲学

### 主动服务，而非被动等待

SMAN 不是用户问什么才答什么，而是：

1. **主动内置足够多的 Skill** - 开箱即用，无需配置
2. **提前做好环境工作** - 工具链预配置，拿来就用
3. **主动记录用户习惯** - 学习用户偏好，越用越顺手
4. **主动引导用户** - 在用户卡住时，主动建议下一步

### 零门槛，双击即用

- 用户双击图标，就可以开始使用
- 不需要懂 AI，不需要懂编程
- SMAN 自己就是专家，帮用户搞定一切

## 核心能力

### 1. 预置 Skill 生态

内置丰富的场景化 Skill，覆盖常见开发任务：

- 代码审查与优化
- 测试用例生成
- 文档编写
- Bug 分析与修复
- 架构设计建议
- 代码重构
- ...

### 2. 环境自配置

- 自动检测开发环境
- 自动配置工具链
- 自动连接必要的 API

### 3. 用户习惯学习

- 记录用户的代码风格偏好
- 记录用户常用的操作模式
- 记录用户的项目上下文
- 越用越懂你

### 4. 主动引导

- 检测到问题时，主动提出解决方案
- 在合适的时机建议下一步行动
- 帮用户发现他们没意识到的问题

## 技术原则

### 简单优先

- 能用 TypeScript 解决的，不引入 Rust
- 能用现有方案的，不造新轮子
- 编译要快，迭代要快

### 本地优先

- 数据在用户本地，隐私有保障
- 不依赖网络也能用
- 启动要快，响应要快

### 渐进增强

- 开箱即用，零配置
- 高级用户可以深度定制
- 插件化扩展能力
