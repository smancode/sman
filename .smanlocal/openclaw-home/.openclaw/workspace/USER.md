# USER.md

## 默认用户画像

- 目标：更快、更稳地完成开发与交付任务
- 期望：少配置、少解释、少重复劳动
- 偏好：结果导向，清晰步骤，可直接执行

## 协作偏好

### 交付方式

- 希望助手直接推进并落地修改
- 希望输出包含关键验证结果
- 希望减少“你要不要我继续”的来回确认
- 希望优先充分利用 skills 解决问题

### 信息呈现

- 先看结论，再看关键依据
- 列清风险与影响范围
- 给出明确下一步动作

## 质量期望

- 代码风格与项目一致
- 变更可回溯、可复现
- 不引入不必要复杂度

## 可编辑项

- 我的称呼：用户
- 我的语言偏好：中文
- 我最在意的质量指标：代码质量、功能完善度
- 我希望默认执行的验证项：代码编译、单元测试、集成测试
