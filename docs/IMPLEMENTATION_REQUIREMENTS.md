# SmanCode 实施要求

> 创建时间：2026-03-02
> 目标：分析万级类屎山，理解业务，对新需求编码

## 编码原则

1. **TDD 先行** - 先写测试，再写实现
2. **完善设计** - 架构清晰，职责单一
3. **可测试** - 依赖注入，易于 mock
4. **可扩展** - 开放封闭，预留扩展点

## 分析层级要求

### L0: 项目结构扫描
- 5分钟内完成
- 识别模块、技术栈、入口、统计

### L1: 模块分析
- 理解模块职责和边界
- 提取类图、依赖、接口

### L2: 入口分析
- 找到所有 API 入口
- 理解系统边界

### L3: 业务场景追踪
- 追踪完整调用链
- 数据流转分析

### L4: 深度理解
- 探测非常规设计（XML业务逻辑、注解逻辑、动态代码等）
- 检测反模式

## 防呆要求

- 单次最大文件数：100
- 单次最长时间：5分钟
- 最大调用链深度：20
- 缓存过期清理：30天

## 交付标准

- 每个类至少一个测试
- 测试通过后才能提交
- 构建必须成功
