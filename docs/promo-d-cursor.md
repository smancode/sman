# D 版 — 对比替代型

> Cursor 很好，但它只帮你写代码

你一天的工作，写代码占多少？三分之一？一半？

剩下的时间在干嘛？读别人的代码、查 bug、写报告、对齐需求、回答同事"这个接口是干嘛的"、纠结 commit message 怎么写。

Cursor 帮不了你这些。

---

**Sman 不只是写代码。**

| 场景 | Cursor | Sman |
|---|---|---|
| 写代码 | 很强 | 很强 |
| 记住项目知识 | 不行 | 你聊过的它都记住，同事也能用 |
| 帮你写 commit message | 不行 | 一键生成 |
| 跑重复任务 | 不行 | 大白话设定时任务 |
| 你走了别人能用吗 | 不能 | 知识跟着 git 走 |
| 在手机上用 | 不行 | 企微、飞书、微信都能聊 |
| 私有部署 | 不行 | 数据不出内网 |
| 开源 | 闭源 | MIT |

**你可以两个一起用。** Cursor 写代码，Sman 管剩下的。

---

Sman 做的最重要的一件事：**你和它讨论过的东西，它都记住了。** 业务逻辑、踩过的坑、设计决策——自动提取成知识，保存到项目里，跟着 git 走。

下次打开它还记得，你同事打开他也知道。

不需要写 wiki，不需要开会同步，聊过就算数。

---

下载 → 填 API Key → 选目录 → 开聊。
