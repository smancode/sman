# Agent 集市像素世界 — 冷色奇幻学院整体视觉设计

> 2026-04-14 | Phase 4 迭代后（解决专家评审 42 个问题）

## 核心定位

**一句话**：Hogwarts 式冷色奇幻学院风格的像素世界地图，用户进入后一眼看到中央广场、四个功能分区、和自己的 Agent 角色，通过视觉层次自然引导探索。

---

## 1. 色彩体系：冷色基调 + 冷调暖色点缀

### 1.1 核心原则（解决美术师 #1, #9 / UX #8）

**不搞纯冷色禁令**。采用"冷色基调 + 冷调点缀色"策略：
- 基调严格冷色（蓝/紫/银/灰）
- 允许低饱和度冷调暖色作为点缀：冷绿植被、冷棕木质、冷粉花朵
- 金色 #FFD700 保留为唯一高饱和暖色，仅用于声望榜和 hover 选中

### 1.2 色彩体系（修正版）

| 层级 | 用途 | 主色 | 明度 | 说明 |
|------|------|------|------|------|
| **边界** | 地图边缘 | `#10141E` | 10% | 极深蓝黑（比原来更暗，给地面留明度空间） |
| **Canvas背景** | 空白区域 | `#10141E` | 10% | 同边界色 |
| **草地** | 可走区域 | `#2A4A42` → `#34584E` | 22-28% | **冷青绿**（非灰蓝），保留"绿"的色相暗示 |
| **路径** | 石板路 | `#4A5E6E` → `#5A6E7E` | 38-45% | 冷灰蓝石板，与草地明度差 ≥15% |
| **广场** | 中央石砖 | `#7B8FA8` → `#95A5B8` | 56-65% | 银灰色，最亮的地面 |
| **水域** | 池塘 | `#2C4E7B` → `#3B5998` | 30-41% | **靛蓝，明度高于草地**，加冰蓝高光像素 |
| **建筑** | 功能建筑 | 见下表 | - | 每种建筑独特主色+轮廓 |
| **强调** | 高亮选中 | `#FFD700`(金) `#A8D8EA`(冰蓝) | 65%/79% | 金色=重要焦点，冰蓝=魔法反馈 |
| **UI文字** | 名字标签 | `#E8E8E8`(银白) | 91% | 主要文字用银白，非冰蓝 |
| **UI强调** | hover/交互 | `#A8D8EA`(冰蓝) | 79% | 仅限交互反馈 |

### 1.3 分区地面色（解决 UX #2：四区差异太小）

| 区域 | 地面色 | 明度 | 色相差异 |
|------|--------|------|---------|
| **北·市场街** | `#35304A` | 22% | 偏紫灰（神秘市场氛围） |
| **中央广场** | `#7B8FA8` | 56% | 银灰（庄重核心） |
| **西·魔法水池** | `#2C4E7B` | 30% | 偏靛蓝（水面区域） |
| **南·任务区** | `#3A4F5F` | 33% | 偏冷灰蓝（实用区域） |
| **东·扩展区** | `#2A4A42` | 22% | 偏冷青绿（基础草地色） |

四区色相分布：紫灰(280°) / 银灰(210°) / 靛蓝(220°) / 灰蓝(200°) / 青绿(170°)，每区至少 20° 色相差。

### 1.4 描边规范（解决美术师 #4）

| 元素 | 描边色 | 描边宽度 | 说明 |
|------|--------|---------|------|
| Agent (32px) | `#1E2A3C` | 1px | 比背景 `#10141E` 稍亮，保证轮廓可见 |
| 建筑 (64-96px) | `#1E2A3C` | 1-2px | 大尺寸可承受 2px |
| 装饰物 | `#1E2A3C` | 1px | 统一 |

关键：描边色 `#1E2A3C`（明度 18%）比背景 `#10141E`（明度 10%）高 8%，在暗背景上能看到轮廓。

### 1.5 对比度梯度（解决美术师 #3）

```
#10141E(10%) → #2A4A42(22%) → #4A5E6E(38%) → #6B4E8D(42%) → #7B8FA8(56%) → #A8D8EA(79%) → #FFD700(65%)
```

每级明度差 ≥12%。金色是特殊强调色，不参与线性梯度。

### 1.6 冰蓝 #A8D8EA 使用限制（解决 UX #11）

仅用于以下 4 种场景，不再承担 11 种功能：
1. **交互反馈** — hover 高亮、点击爆发
2. **灯笼光晕** — 魔法灯笼脉动
3. **水面波光** — 池塘反光
4. **状态气泡** — Agent idle 状态

建筑标签文字改用 `#E8E8E8` 银白。环境粒子改用 `#C8E8F4` 浅冰蓝。

## 3. 建筑视觉

### 3.1 建筑区分度（解决美术师 #5）

每种建筑必须有 **独特剪影轮廓**，不能只靠颜色区分：

| 建筑 | 主色 | 辅色 | 剪影特征 |
|------|------|------|---------|
| **摊位** | `#6B4E8D` 紫 | 每个摊位不同色帘幔(冰蓝/紫/银白/深蓝交替) | 尖顶帐篷 |
| **声望榜** | `#E8C460` 柔金 | `#7B8FA8` 银灰 | 竖长碑形 + 金冠 |
| **悬赏板** | `#4A5568` 纯冷灰 | `#8B9A9A` 浅灰 | 宽矮矩形，无紫色 |
| **搜索站** | `#4A6FA5` 偏蓝 | `#7BC0D4` 深冰蓝(非 #A8D8EA) | 圆球顶 |
| **工坊** | `#6B5A7A` 偏紫灰 | `#B0C0D0` 浅银蓝(非 #A8D8EA) | 方形 + 烟囱 |

关键变化：
- 悬赏板去掉紫色，改用纯冷灰，与摊位区分
- 搜索站和工坊不再共用冰蓝高光，各自用不同深浅的蓝/银
- 声望榜用柔金 `#E8C460`（饱和度 75%），比纯金 `#FFD700` 柔和

### 3.2 建筑标签（解决 UX #3, #12）

- **摊位区**：6 个摊位不在各自显示标签，改为在市场街入口放一个区域标牌 "市场街"
- 摊位 hover 时才显示具体标签
- **功能建筑**：声望榜/悬赏板/搜索站/工坊 各自常驻标签
- 标签格式：emoji + 中文名
- 标签背景：不透明 `#1A2332`（非半透明，避免被建筑颜色穿透）
- 标签文字：`#E8E8E8` 银白（非冰蓝）
- 标签用 OffscreenCanvas 缓存，每帧 drawImage 一次

## 9. 技术实现方案

### 9.1 "只换色"修正为"提取+换色+小改逻辑"（解决前端 #1, #7, #9）

**不是"只改颜色"**，实际工作量：

1. **提取阶段**：将 35+ 处硬编码颜色全部提取到 palette.ts 新增常量
2. **换色阶段**：替换 palette.ts 中所有常量为冷色系
3. **逻辑微调**：
   - 草地绘制：保留绿色色相的冷青绿（#2A4A42），删除暖绿草丛点缀，改为冷蓝微光点
   - 水面绘制：明度提升（#2C4E7B），加冰蓝高光像素
   - 花丛绘制：3 色改为冷粉+冰蓝+紫
   - 分区地面：ProceduralAssets 按瓦片坐标选择分区色
   - 声望榜红宝石：改为紫色宝石
   - 摊位物品：三种不同冷色（蓝/紫/金）

### 9.2 水面动画方案（解决前端 #4）

采用**方案 B**：在 WorldRenderer 主循环中，地面 drawImage 之后，对水面区域叠加半透明闪烁层。不修改 TileMap 缓存机制。

- 水面瓦片坐标已知（map-data.ts 中 tileId=3 的位置）
- 每帧对水面瓦片区域画半透明冰蓝矩形，透明度随 sin(frame) 变化
- 开销极低（~10-15 个瓦片区域）

### 9.3 灯笼光晕合成模式（解决前端 #5）

```typescript
ctx.globalCompositeOperation = 'screen';
ctx.fillStyle = `rgba(168,216,234, ${glow})`;
// 画光晕圆
ctx.globalCompositeOperation = 'source-over'; // 恢复
```

### 9.4 建筑标签缓存（解决前端 #3）

每个建筑首次创建时，生成标签 OffscreenCanvas（背景+emoji+中文名），后续每帧 drawImage。避免每帧 fillText。

### 9.5 文件修改范围（修正版）

| 文件 | 修改类型 | 具体内容 |
|------|---------|---------|
| `palette.ts` | 重写 | 所有颜色常量 + 新增 35+ 个命名常量 |
| `ProceduralAssets.ts` | 提取+换色+微调逻辑 | 硬编码颜色提取到 palette，改草地/水面/花丛绘制，加分区地面色逻辑 |
| `WorldRenderer.ts` | 提取+换色+新增标签缓存 | UI 颜色提取+替换，新增建筑标签 OffscreenCanvas 缓存，水面闪烁 pass，hover 改用 screen 合成，脉冲改白色 |
| `DecoLayer.ts` | 提取+换色+改光晕 | 颜色提取+替换，光晕改 screen 合成+提高透明度 |
| `ParticleLayer.ts` | 换色 | 金色粒子改冰蓝+紫 |
| `AgentEntity.ts` | 提取 | `shirtColor: '#41a6f6'` 提取到 palette |
| `WorldSync.ts` | 提取 | `shirtColor: '#41a6f6'` 提取到 palette |
| `WorldCanvas.tsx` | 小改 | Canvas 背景色改为 #10141E，加 cursor:pointer |

## 10. 深度设计：专家评审问题与解决方案

### 问题 1：草地非绿色导致"可走区域"认知失败

### 问题 2：水面与草地明度差 3% 不可区分
**发现者**：像素美术师 | **解法**：水面明度提升到 30%（#2C4E7B），草地保持 22%，差 8%

**发现者**：像素美术师 | **解法**：重新设计梯度，确保每级 ≥12%，金色不参与线性梯度

## MVP 范围

1. palette.ts 重写（全部冷色系 + 新增 35+ 命名常量）
2. ProceduralAssets.ts 提取硬编码颜色 + 冷色替换 + 分区地面色 + 草地/水面/花丛绘制微调
3. WorldRenderer.ts UI 颜色替换 + 建筑标签缓存 + 水面闪烁 pass + hover screen 合成 + 脉冲改白色 + cursor:pointer
4. DecoLayer.ts 颜色替换 + 光晕 screen 合成
5. ParticleLayer.ts 粒子颜色替换
6. AgentEntity.ts / WorldSync.ts 默认颜色提取到 palette
7. WorldCanvas.tsx Canvas 背景色

**不在 MVP 中**：
- 缩放机制
- 键盘快捷键
- 3 种粒子类型（仅保留冰晶飘落）
- Agent 空闲行为动画
- 地图布局变更
- PixelLab PNG 素材生成
- Agent 外观自定义
- 东部扩展区装饰（后续迭代）
