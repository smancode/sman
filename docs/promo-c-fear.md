# C 版 — 恐惧驱动型

> 你在这个项目待了三年，离职那天，知识就清零了

你脑子里的东西比代码值钱。哪些接口有坑、哪些设计是历史包袱、哪些逻辑千万别动——这些全在你脑子里。

你走了，新人来，从零开始猜。猜对了运气好，猜错了线上事故。

**你每天回答新人"这个为什么这么写"的问题，就是在当活文档。** 你自己的活干不完，还得帮别人读代码。

---

Sman 干的事就是：**把你脑子里的东西变成文字，而且不需要你专门写。**

你和 AI 讨论项目问题的时候，它在旁边听着，自动把业务规则、设计决策、编码约定提取出来，存到项目目录里。

你不用写文档。聊过就算。

而且这些知识跟着 git 走。你同事 pull 了代码，他的 AI 也知道了。不需要开会同步，不需要维护 wiki。

```
你: 这个缓存为什么用 Redis 不用本地？

Sman: 多实例部署，本地缓存会不一致。去年架构评审定的。

→ 这个知识点自动保存到项目里
→ 同事 pull 代码后他的 AI 也知道了
→ 不会有人再问一遍同样的问题
```

---

你在这个项目踩过的坑、做过的决定、积累的经验——别让它们只活在你脑子里。

试一下，不要钱。
