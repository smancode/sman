## 任务 2.1：实现空白发现机制

### 做什么
在 `domain/puzzle/` 下实现 4 种空白发现：

```kotlin
// GapDetector.kt
class GapDetector(
    private val puzzleStore: PuzzleStore,
    private val projectPath: String
) {

    // 方式一：文件变更触发
    fun detectByFileChange(changedFiles: List<String>): List<Gap>

    // 方式二：交叉验证发现
    fun detectByCrossValidation(): List<Gap>

    // 方式三：引用追踪发现
    fun detectByReferenceTracing(): List<Gap>

    // 方式四：用户查询触发
    fun detectByUserQuery(query: String): List<Gap>

    // 综合发现
    fun detectAll(): List<Gap>
}

data class Gap(
    val type: GapType,
    val puzzleType: PuzzleType,
    val description: String,
    val priority: Double,
    val relatedFiles: List<String>,
    val detectedAt: Instant
)

enum class GapType {
    MISSING,         // 缺失
    OUTDATED,        // 过时
    INCONSISTENT,    // 不一致
    INCOMPLETE       // 不完整
}
```

### 为什么
自迭代的核心是发现"不知道什么"，然后填补空白。

// TaskScheduler.kt
class TaskScheduler {

    // 计算任务优先级
    fun calculatePriority(gap: Gap, context: SchedulingContext): Double {
        val roi = calculateROI(gap)
        val freshness = calculateFreshness(gap)
        val heat = calculateHeat(gap)

        return roi * 0.5 + freshness * 0.3 + heat * 0.2
    }

    // 对空白列表排序
    fun prioritize(gaps: List<Gap>): List<Gap>

    // 选择下一个要执行的任务
    fun selectNext(gaps: List<Gap>, budget: TokenBudget): Gap?
}

    val recentFileChanges: List<String>,

)

## 任务 2.4：实现自迭代循环

### 做什么
整合所有组件，实现主循环：

```kotlin
// PuzzleCoordinator.kt
class PuzzleCoordinator(
    private val store: PuzzleStore,
    private val gapDetector: GapDetector,
    private val scheduler: TaskScheduler,
    private val backgroundScheduler: BackgroundScheduler,
    private val analyzer: PuzzleAnalyzer
) {

    // 主循环
    suspend fun runCycle(): CycleResult {
        // 1. 发现空白
        val gaps = gapDetector.detectAll()

        // 2. 优先级排序
        val prioritized = scheduler.prioritize(gaps)

        // 3. 选择任务
        val task = scheduler.selectNext(prioritized, budget)

        // 4. 执行分析
        val result = analyzer.analyze(task)

        // 5. 验证结果
        val validated = validator.validate(result)

        // 6. 更新拼图
        store.save(validated)

        // 7. 返回结果，可能发现新空白
        return CycleResult(...)
    }

    // 启动自迭代
    fun startSelfIterating()

    // 暂停/恢复
    fun pause()
    fun resume()
}
```

### 为什么
这是整个自迭代系统的核心协调器。

### 怎么检查结果
