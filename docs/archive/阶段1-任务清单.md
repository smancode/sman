## 任务 1.1：设计 Puzzle 数据结构

### 做什么
在 `shared/model/` 下创建 Puzzle 相关数据类：

```kotlin
// PuzzleType.kt - 拼图类型枚举
enum class PuzzleType {
    STRUCTURE,      // 结构拼图
    TECH_STACK,     // 技术拼图
    API,            // 入口拼图
    DATA,           // 数据拼图
    FLOW,           // 流程拼图
    RULE            // 规则拼图
}

// PuzzleStatus.kt - 拼图状态枚举
enum class PuzzleStatus {
    PENDING,        // 待分析
    IN_PROGRESS,    // 分析中
    COMPLETED,      // 已完成
    FAILED          // 失败
}

// Puzzle.kt - 拼图块模型
data class Puzzle(
    val id: String,
    val type: PuzzleType,
    val status: PuzzleStatus,
    val content: String,           // Markdown 内容
    val completeness: Double,      // 完成度 0-1
    val confidence: Double,        // 置信度 0-1
    val lastUpdated: Instant,
    val filePath: String           // 存储路径
)
```

### 为什么
为后续拼图系统提供统一的数据模型，确保存储格式一致。

### 怎么检查结果
```bash
# 检查文件存在
ls src/main/kotlin/com/smancode/sman/shared/model/Puzzle*.kt

# 编译通过
./gradlew compileKotlin

# 单元测试
./gradlew test --tests "*Puzzle*"
```

```kotlin
// PuzzleStore.kt
class PuzzleStore(private val projectPath: String) {

    // 保存拼图到 Markdown 文件
    fun save(puzzle: Puzzle): Result<Unit>

    // 加载拼图
    fun load(puzzleId: String): Result<Puzzle?>

    // 加载所有拼图
    fun loadAll(): Result<List<Puzzle>>

    // 删除拼图
    fun delete(puzzleId: String): Result<Unit>
```

## 任务 1.5：改造 6 种分析类型为 Puzzle

### 做什么
将现有的 6 种分析类型改造为 Puzzle 格式：

| 原类型 | 新 PuzzleType | 输出文件 |
|--------|--------------|----------|
| PROJECT_STRUCTURE | STRUCTURE | PUZZLE_STRUCTURE.md |
| TECH_STACK | TECH_STACK | PUZZLE_TECH.md |
| API_ENTRIES | API | PUZZLE_API.md |
| DB_ENTITIES | DATA | PUZZLE_DATA.md |
| - | FLOW | PUZZLE_FLOW_*.md |
| - | RULE | PUZZLE_RULES.md |

修改 `AnalysisLoopExecutor` 和 `ProjectAnalysisLoop`，使其输出 Puzzle 而不是原来的格式。

### 为什么
