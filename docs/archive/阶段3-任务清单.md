## 任务 3.1：设计用户偏好数据结构

### 做什么
在 `shared/model/` 下创建偏好相关数据类：

```kotlin
// UserPreference.kt
data class UserPreference(
    val id: String,
    val type: PreferenceType,
    val key: String,              // 偏好键（如 "naming_convention"）
    val value: String,            // 偏好值（如 "camelCase"）
    val confidence: Double,       // 置信度
    val source: PreferenceSource, // 来源
    val learnedAt: Instant,
    val usageCount: Int           // 使用次数
)

enum class PreferenceType {
    CODING_STYLE,       // 编码风格
    NAMING_CONVENTION,  // 命名规范
    ARCHITECTURE,       // 架构偏好
    FRAMEWORK,          // 框架偏好
    BUSINESS_RULE       // 业务规则
}

enum class PreferenceSource {
    EXPLICIT,           // 用户明确指定
    IMPLICIT,           // 从行为推断
    CORRECTION          // 从修正中学习
}

// Feedback.kt
data class Feedback(
    val id: String,
    val type: FeedbackType,
    val content: String,
    val context: String,          // 触发反馈的上下文
    val timestamp: Instant
)

enum class FeedbackType {
    POSITIVE,           // 正面反馈
    NEGATIVE,           // 负面反馈
    CORRECTION,         // 修正
    CLARIFICATION       // 澄清
}
```

### 为什么
结构化的偏好数据是学习系统的基础。

## 任务 3.4：实现偏好注入器

### 做什么
在 `domain/memory/` 下实现偏好注入到 Prompt：

```kotlin
// PreferenceInjector.kt
class PreferenceInjector(
    private val memoryStore: MemoryStore
) {

    // 注入偏好到 System Prompt
    fun injectIntoSystemPrompt(basePrompt: String): String {
        val preferences = memoryStore.load()?.preferences ?: emptyList()

        val preferenceSection = buildPreferenceSection(preferences)

        return "$basePrompt\n\n$preferenceSection"
    }

    // 构建偏好章节
    private fun buildPreferenceSection(preferences: List<UserPreference>): String {
        return buildString {
            append("## 用户偏好\n\n")
            preferences.groupBy { it.type }.forEach { (type, prefs) ->
                append("### ${type.displayName}\n")
                prefs.forEach { pref ->
                    append("- ${pref.key}: ${pref.value}\n")
                }
            }
        }
    }

    // 按相关度筛选偏好
    fun filterRelevantPreferences(
        preferences: List<UserPreference>,
        context: String
    ): List<UserPreference>
}
```

### 为什么
将学到的偏好应用到后续交互中。

## 任务 3.5：实现记忆刷新服务

### 做什么
在 `domain/memory/` 下实现定期刷新：

```kotlin
// MemoryFlushService.kt
class MemoryFlushService(
    private val memoryStore: MemoryStore,
    private val feedbackCollector: FeedbackCollector,
    private val preferenceExtractor: PreferenceExtractor
) {

    // 刷新记忆（处理待处理的反馈）
    suspend fun flush(): FlushResult {
        val pendingFeedbacks = feedbackCollector.getPendingFeedbacks()

        val newPreferences = mutableListOf<UserPreference>()
        for (feedback in pendingFeedbacks) {
            val extracted = preferenceExtractor.extractFromFeedback(feedback)
            newPreferences.addAll(extracted)
            feedbackCollector.markProcessed(feedback.id)
        }

        val memory = memoryStore.load() ?: ProjectMemory.empty()
        val updated = memory.mergePreferences(newPreferences)
        memoryStore.save(updated)

        return FlushResult(
            feedbacksProcessed = pendingFeedbacks.size,
            newPreferencesAdded = newPreferences.size
        )
    }

    // 定期刷新
    fun startPeriodicFlush(intervalMs: Long = 60_000)  // 1 分钟

    // 停止定期刷新
    fun stopPeriodicFlush()
}

data class FlushResult(
    val feedbacksProcessed: Int,
    val newPreferencesAdded: Int
)
```

### 为什么
批量处理反馈，避免频繁调用 LLM。

### 怎么检查结果
