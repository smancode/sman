## 一、设计原则

### 1.1 核心原则

| 原则 | 说明 |
|------|------|
| **单一职责** | 一个文件只做一件事 |
| **行数限制** | 单文件不超过 300 行（Kotlin） |
| **目录限制** | 单目录不超过 8 个文件/子目录 |
| **扁平优于嵌套** | 减少目录层级，最多 3 层 |
| **按领域划分** | 按业务领域组织，而非技术层 |

### 1.2 命名规范

| 规范 | 示例 |
|------|------|
| 目录用单数 | `model/` 不用 `models/` |
| 服务用 `*Service` | `AnalysisService` |
| 仓库用 `*Repository` | `PuzzleRepository` |
| 工具用 `*Tool` | `ReadFileTool` |
| 协调器用 `*Coordinator` | `PuzzleCoordinator` |

## 六、风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| 大规模重构影响稳定性 | 分阶段进行，每阶段验证 |
| import 大量修改 | 使用 IDE 重构功能 |
| 测试覆盖不足 | 重构前先写测试 |
| 文档不同步 | 同步更新 README.md 和 CLAUDE.md |
