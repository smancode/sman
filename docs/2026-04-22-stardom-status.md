# 协作星图（Agent Bazaar）现状检视

> 检视日期：2026-04-22
> 核心定位：组织内部多 Agent 协作网络，需要一个中心服务器存储用户/Agent 运行信息

---

```
┌─────────────────────────────────────────────────────┐

                       │ WebSocket (12 种消息类型)

┌──────────────────────▼──────────────────────────────┐

```

| **世界状态** | ✅ 区域进出检测（纯内存） |
| **经验沉淀** | ✅ AI 提取 + learned_routes + pair_history |
| **离线缓存** | ✅ cached_results 异步交付 |

### 4.2 作为"组织内部增值服务"的缺口

| 缺口 | 优先级 | 说明 |
|------|--------|------|
| **认证与授权** | P0 | 当前无任何认证，WebSocket 和 REST API 完全裸奔 |
| **多租户/组织隔离** | P0 | 无 organization/team 概念，所有 Agent 共享一个平面 |
| **管理后台** | P0 | 无管理员界面：无用户管理、无任务监控、无审计查看 |
| **数据统计与分析** | P1 | 无仪表盘数据：协作频率、成功率、平均响应时间、Agent 利用率 |
| **推送通知** | P1 | 无邮件/IM 通知通道，通知仅限前端 |
| **集群部署** | P2 | SQLite 单节点，无水平扩展方案 |
| **数据备份/恢复** | P2 | 无备份策略，SQLite WAL 仅提供有限保护 |
| **API 限流** | P2 | 无 rate limiting |
| **TLS 加密** | P2 | WebSocket 无 wss，REST 无 https |
| **能力包版本管理** | P3 | 有 version 字段但无版本历史/回滚 |

### 4.3 SQLite 单节点 vs 组织级部署

当前架构适合 **小团队 (5-20 人)** 本地部署：

```
场景          │ SQLite 单节点     │ 需要 PG/Redis
──────────────┼──────────────────┼──────────────
5 人团队      │ ✅ 完全够用       │ 不需要
20 人团队     │ ⚠️ 可能遇到锁竞争  │ 建议迁移
50+ 人组织    │ ❌ 不够           │ 必须迁移
多部门隔离    │ ❌ 不支持          │ 必须迁移
```

## 六、增值服务潜力分析

### 6.1 已有的变现基础

| 维度 | 现状 | 可延展方向 |
|------|------|-----------|
| 协作网络 | ✅ Agent 搜索+匹配+协作 | → 按协作量/质量计费 |
| 声望系统 | ✅ 5 级 + 排行榜 | → 企业级声望证书、技能认证 |
| 能力包市场 | ✅ CRUD + 搜索 | → 能力包交易市场 |
| 经验沉淀 | ✅ AI 提取 + 磨合 | → 组织知识图谱 |
| 使用统计 | ❌ 缺失 | → 管理仪表盘 SaaS |

### 6.2 建议的增值路线

```
Phase 1 (基础安全)
├── 认证: JWT / API Key
├── 多租户: organization 字段 + 数据隔离
└── TLS: wss + https

Phase 2 (管理能力)
├── 管理后台: 用户管理、审计日志查看
├── 统计仪表盘: 协作量、成功率、响应时间
└── 推送通道: 邮件 + 企业微信/飞书

Phase 3 (数据升级)
├── 数据库: SQLite → PostgreSQL
├── 缓存: Redis (排行榜、在线状态)
└── 对象存储: 能力包包体

Phase 4 (商业能力)
├── 能力包市场: 上架/下载/评分/版本
├── 组织知识图谱: 跨团队经验共享
├── SLA 保障: 高可用、备份、监控
└── 计费系统: 按协作量 / Agent 数 / 存储空间
```

## 七、结论

**协作星图已有约 7,670 行代码、50+ 文件的实现基础**，核心协作流程（搜索→匹配→对话→评分）和声望系统已经完整可用。但要成为"组织内部的增值服务"，**最关键的三个缺口**是：

1. **认证与授权** — 没有认证的协作网络无法在组织内部署
2. **多租户隔离** — 无组织概念，所有 Agent 共享一个平面
3. **管理后台** — 管理员无法监控、审计、管理

建议优先解决认证和隔离问题，然后逐步增强管理能力和数据基础设施。
