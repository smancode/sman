# A 版 — 痛点共鸣型

> 别再自己踩别人踩过的坑了

昨天你花一下午定位一个 bug，最后发现半年前有人已经踩过，当时还跟同事聊过——但没人告诉你。

**入职接手老项目，代码看不懂。** 注释写着"TODO: 优化"——2021 年写的。问了一圈没人说得清，你硬着头皮改了，上线出了事背锅的是你。

**同一个 bug，不同的人各踩一遍。** 上次排查的结论在企微群里，早就沉了。每次都从零开始。

**同事离职，他负责的模块成了黑洞。** 没人敢动，新来的人纯靠猜。

**每周的项目报告，抄上周的改改数字。** 没人看，但你不能不交。

---

**Sman 做的事很简单：帮你记住。**

你和 AI 讨论过的业务逻辑、踩过的坑、做过的设计决策，它会自动提取成知识，保存到项目目录里，跟着 git 走。

下次你打开，它还记得。你同事打开，他也知道。人走了，笔记还在。

打开 Sman，选项目目录，直接问：

```
你: 这个退款流程会影响库存锁吗？

Sman: 部分发货状态下锁不会释放，去年防超卖加的。
      代码在 RefundService.ts:142。
```

这些可能是你前任半年前聊的。不用问人，不用翻文档。
