# 意图识别提示词

## 任务说明

分析用户的输入，判断用户的意图类型。

## 意图类型

### 1. CHAT（闲聊）
**特征**：
- 问候语：你好、在吗、hi
- 闲聊：天气怎么样、你是谁、你能做什么
- 无具体代码相关需求

**示例**：
- "你好"
- "你能做什么"
- "今天天气怎么样"

### 2. WORK_ANALYZE（代码分析）
**特征**：
- 包含代码相关的关键词：搜索、查找、分析、理解、看看
- 提及具体的功能、模块、业务概念
- 需要调用工具来分析代码

**示例**：
- "搜索放款逻辑"
- "分析调用链"
- "找出 PaymentService 的用途"
- "理解订单处理流程"

### 3. WORK_CODE（代码操作）
**特征**：
- 包含代码修改相关的关键词：添加、修改、删除、重构
- 明确的代码操作指令
- 需要修改代码文件

**示例**：
- "帮我在 PaymentService 中添加一个日志"
- "修改 OrderController 的错误处理"
- "重构 validate 方法"

## 输出格式

请输出 JSON 格式：

```json
{
  "intent": "CHAT | WORK_ANALYZE | WORK_CODE",
  "confidence": 0.95,
  "reasoning": "判断理由"
}
```

## 判断标准

- **confidence**：0.0 ~ 1.0，表示判断的置信度
- **reasoning**：简要说明判断依据（1-2 句话）

## 注意事项

- 当意图不明确时，选择置信度最高的类型
- 当同时符合多个意图时，优先选择 WORK_CODE > WORK_ANALYZE > CHAT
- reasoning 使用中文
